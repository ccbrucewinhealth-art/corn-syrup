## React 對應選型

| 參考系統技術 | React 對應 | 選擇理由 |
|---|---|---|
| Vue 3 SFC | React + TypeScript + TSX | 保留元件、頁面、layout 的一對一對應，改以 function component 與 props 呈現。 |
| Vue Router 4 | `react-router-dom` | 支援巢狀 route、動態參數、fallback route 與 SPA 導覽。 |
| Vue I18n | `i18next` + `react-i18next` | 支援 `zh-TW`、`en`、`ja`，預設 `zh-TW`，符合本系統 i18n 要求。 |
| Vite + Vue plugin | Vite + `@vitejs/plugin-react` | 延續參考系統的快速開發、建置與 preview 流程。 |
| Vue mixins | React hooks/helpers | 將日期、語系、行動裝置、公開頁、Socket、Theme 共用行為轉為 hook/helper。 |
| Bootstrap / SCSS | Bootstrap CSS + React className | 保留原 UI class 語意，降低視覺差異。 |
| Socket.IO Client | `socket.io-client` | 保留即時事件協定與後端相容性。 |
| Axios | `axios` | 保留 API 呼叫模式。 |
| Chart.js / vue-chartjs | Chart.js + `react-chartjs-2` | 對應 `PingChart` 等圖表元件。 |
| 動態 import | React `lazy` + route lazy boundary | 對應參考系統設定頁與通知頁延遲載入行為。 |

## 轉換原則

1. Vue `.vue` 元件與頁面以一對一檔案名稱轉為 React `.tsx`。
2. Vue `.js`/`.ts` 共用模組依語意轉為 TypeScript，保留來源檔案職責。
3. 靜態樣式與資產檔不轉換程式語言，於檢查表標示 `DoTranslate=NO`，必要時直接複製或引用。
4. 單一檔案接近 500 行時，依功能切分到有意義命名的子檔案，不使用 `part+no` 命名。
5. 若來源屬網頁使用 view 或前端程式，轉為 React；若屬純資產、語言資料或目錄占位，僅標記不轉換。

## 階段切分

1. 核心殼層與路由：`App.vue`、`main.js`、`router.js`、`layouts/*`。
2. 監控儀表板與頁面：`pages/*` 與監控清單/細節相關 components。
3. 共用元件與設定：通用表單、dialog、settings components、mixins、utils。
4. 通知與即時互動：`components/notifications/*`、Socket 與通知設定表單。
5. i18n、資產與建置交付：`i18n.js`、`lang/*`、assets、shell、Makefile、離線套件。
